# Izbushka Web Core

Центральный узел управления робототехническим комплексом Izbushka. Система реализована на базе асинхронного фреймворка FastAPI и обеспечивает координацию между аппаратным уровнем, пользовательскими интерфейсами и внешними ИИ-сервисами.

## Технологическая архитектура

Проект перенесен с архитектуры [flask_template](https://github.com/Emeteil/flask_template) на [fastapi_template](https://github.com/Emeteil/fastapi_template), что обеспечило автоматическую генерацию OpenAPI-документации (Swagger), строгую типизацию данных и повышенную производительность асинхронной обработки WebSocket-соединений.

### Основные компоненты

1.  **Транспортная шина (TransportBus)**: Абстракция транспортного уровня для прозрачного переключения между физическим устройством и симулятором.
    *   **COM-LINK-RT**: Бинарный протокол реального времени для связи с контроллером по Serial-интерфейсу. Использует зависимости из репозитория [com_link_rt](https://github.com/Emeteil/com_link_rt_client/tree/robot).
    *   **VirtualLink**: TCP-канал для взаимодействия с Unity-симулятором [izbushka_simulation](https://github.com/Emeteil/izbushka_simulation).
2.  **Оптимизация управления (control.js)**: Алгоритм адаптивного троттлинга команд джойстика. Частота отправки пакетов динамически меняется в зависимости от вектора движения и изменения скорости (скоростные зоны), что минимизирует нагрузку на Serial-канал при сохранении высокой отзывчивости.
3.  **Медиа-сервер**: Реализация MJPEG-стриминга видеопотока. Стрим является основным источником визуальных данных для [izbushka-voice-interface](https://github.com/Emeteil/izbushka-voice-interface).
4.  **API Эмоций**: Интерфейс управления анимацией лицевой панели робота.

## Сетевая конфигурация

*   **Порт 80 (HTTP/WS)**: Основной API, веб-интерфейс и Socket.IO шлюз.
*   **Порт 5470 (TCP)**: Прием входящих соединений от Unity-симулятора.

## Зависимости и сабмодули

Проект включает в себя или взаимодействует со следующими компонентами:
*   [com_link_rt](https://github.com/Emeteil/com_link_rt_client/tree/robot) - клиентская Python-библиотека протокола.
*   [COM-LINK-RT](https://github.com/Emeteil/COM-LINK-RT/tree/robot) - низкоуровневая реализация протокола для STM32/Arduino.
